Manual de Git: Conceptos Básicos

En esta entrada, exploraremos algunos de los conceptos básicos de Git, como la inicialización de un repositorio, la creación de ramas, y la conexión de tu repositorio local con un repositorio remoto en GitHub.

1. Inicialización del repositorio localmente

El primer paso para empezar a trabajar con Git es crear un repositorio local en tu máquina. Esto te permitirá rastrear los cambios de los archivos de tu proyecto.

Para inicializar un nuevo repositorio, puedes utilizar el siguiente comando:

git init

Este comando crea una carpeta oculta llamada .git en tu directorio de trabajo, que es donde Git almacena toda la información del historial de cambios.

Después de esto, puedes comenzar a añadir archivos y realizar commits para guardar los cambios de tu proyecto.

2. Creación de branches

En Git, las ramas (branches) te permiten trabajar en diferentes características o partes de tu proyecto de forma aislada. El uso de ramas facilita la colaboración y te ayuda a probar cambios antes de fusionarlos con el proyecto principal.

Para crear una nueva rama, puedes utilizar el siguiente comando:

git branch entradablog-git

Para cambiar a la nueva rama creada:

git checkout entradablog-git

Una vez hayas realizado tus cambios en la nueva rama, puedes fusionar estos cambios con la rama principal (master o main) usando el comando merge:

git checkout main
git merge entradablog-git

Pantallazo del proceso de creación y fusión de ramas:

Las ramas son útiles para trabajar en diferentes características sin afectar el código principal, permitiendo una mayor flexibilidad en el desarrollo del proyecto.

3. Agregar un repositorio remoto

Una vez que hayas trabajado localmente en tu proyecto, puedes conectar tu repositorio local a uno remoto, como GitHub, para colaborar con otros desarrolladores y tener una copia segura de tu proyecto en la nube.

Para agregar un repositorio remoto, primero debes tener una cuenta en GitHub y crear un repositorio vacío. Luego, usa el siguiente comando para enlazar tu repositorio local con el remoto:

git remote add origin https://github.com/afel2312/page_blog_personal.git

Este comando enlaza tu repositorio local con el repositorio remoto en GitHub. Luego puedes empujar tus cambios con el siguiente comando:

git push -u origin main

A partir de este punto, puedes colaborar en GitHub y seguir haciendo commits y pushes para mantener tu código actualizado.
